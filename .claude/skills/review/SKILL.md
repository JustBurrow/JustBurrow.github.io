블로그 포스트 리뷰 워크플로우의 오케스트레이터다. 포스트 내용을 직접 분석해 어떤 직원(서브에이전트)이 필요한지 판단하고, 적절한 순서로 위임해 통합 결과를 보고한다.

## 요구사항

### 포스트 분석

스킬은 반드시 에이전트 실행 전에 `Read` 도구로 포스트 파일 전체를 읽고 다음 속성을 파악해야 한다:

- **사실 주장 밀도**: 검증 가능한 연도·버전·역사적 사건·인과관계 주장의 수
- **의견·예측 비율**: 주장 중 저자의 의견·예측인 항목의 비율
- **제3자 언급**: 실명·소속·발언 등 타인 정보의 존재 여부
- **민감 자료 포함**: 코드 스니펫, URL, 자격증명 형식 문자열의 존재 여부

### 에이전트 선택

스킬은 포스트 내용에 관계없이 반드시 [`security-reviewer`](../../agents/security-reviewer.md)를 실행해야 한다.

포스트에 검증 가능한 사실 주장(연도, 버전, 역사적 사건, 인과관계)이 포함된 경우, 스킬은 반드시 [`fact-checker`](../../agents/fact-checker.md)를 실행해야 한다.

포스트에 구조화된 논증이나 주장(thesis)이 있는 경우(메모, 일기, 링크 모음 제외), 스킬은 반드시 [`reviewer`](../../agents/reviewer.md)를 실행해야 한다.

[`fact-checker`](../../agents/fact-checker.md)와 [`reviewer`](../../agents/reviewer.md) 모두 불필요한 경우, 스킬은 반드시 그 이유를 명시해야 한다.

필요한 에이전트가 [`.claude/agents/`](../../agents/)에 없는 경우, 스킬은 반드시 채용(에이전트 파일 생성)한 뒤 실행해야 한다.

### 실행 순서

스킬은 반드시 다음 순서로 에이전트를 실행해야 한다:

1. [`security-reviewer`](../../agents/security-reviewer.md) — 먼저 실행. 🔴 차단 항목이 있으면 즉시 사용자에게 보고하고 명시적 지시를 기다린다. 지시 없이 나머지 에이전트를 실행해서는 안 된다.
2. [`fact-checker`](../../agents/fact-checker.md) — 해당하는 경우 실행.
3. [`reviewer`](../../agents/reviewer.md) — 해당하는 경우 실행.

### 에이전트 간 컨텍스트 전달

[`reviewer`](../../agents/reviewer.md)를 실행하기 전에, 스킬은 반드시 [`fact-checker`](../../agents/fact-checker.md) 결과에서 다음 항목을 추출해야 한다:

- ⚠️ 부정확 항목: 해당 주장이 논증에서 차지하는 역할 (핵심 근거인지 부수적 예시인지)
- ❓ 확인 불가 항목: [`reviewer`](../../agents/reviewer.md)가 근거 적절성 평가 시 "불확실한 근거"로 처리하도록 명시

스킬은 반드시 추출한 컨텍스트를 다음 형식으로 [`reviewer`](../../agents/reviewer.md) 호출 프롬프트에 포함해야 한다:

```
다음은 fact-checker 결과 중 논리 평가에 영향을 줄 수 있는 항목입니다:

- [부정확 또는 확인 불가 주장] — 논증에서의 역할: [핵심 근거 / 부수적 사례 / 배경 설명]
```

스킬은 [`security-reviewer`](../../agents/security-reviewer.md)의 결과를 다른 에이전트에 전달해서는 안 된다.

### 결과 보고

스킬은 반드시 통합 결과를 사용자에게 보고한 뒤, 수정 지시를 받기 전까지 포스트를 수정해서는 안 된다.

스킬은 여러 에이전트의 출력을 사용자 검토 없이 하나의 수정에 일괄 반영해서는 안 된다.

스킬은 반드시 다음 형식으로 통합 결과를 반환해야 한다:

```markdown
## 리뷰 종합

**파일**: _posts/YYYY-MM-DD-title.md
**실행한 에이전트**: [목록]
**생략한 에이전트**: [목록 및 이유]

### 보안 리뷰 요약

[security-reviewer 결과 요약. 🔴/🟡/🔵 건수 및 핵심 항목]

### 팩트체크 요약

[`fact-checker`](../../agents/fact-checker.md) 결과 요약, 또는 생략 이유
[`reviewer`](../../agents/reviewer.md)에 전달한 컨텍스트 항목

### 논리·근거 리뷰 요약

[reviewer 결과 요약, 또는 생략 이유]

### 종합 판정 및 권고

| 항목 | 상태 |
|------|------|
| 보안 | 발행 가능 / 수정 필요 / 차단 |
| 사실 정확성 | 이상 없음 / 수정 권고 / 미실행 |
| 논리 구조 | 이상 없음 / 개선 권고 / 미실행 |

**다음 단계**: [수정 후 재검토 / PR 진행 가능 / 사용자 판단 필요 항목 목록]
```
